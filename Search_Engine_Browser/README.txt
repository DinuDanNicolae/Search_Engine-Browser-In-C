       Task 1 - Implementarea acestei probleme a durat aproximativ 8 ore. Programul foloseste un tip de date neomogen pentru a retine si a stoca datele despre site-uri: URL, numarul de vizulalizari, numarul de octeti, checksum-ul, titlul, continutul si codul. Pentru rezolvarea task-ului au fost folosite mai multe functii: cele doua functii de initializare(init, init2), functia de citire a codului html (cd) si cea de convertire a culorii. Continutul site-urilor este parsat si stocat cu ajutorul a doi pointeri pentru citirea intre tag-urile de tip html. De asemenea, programul se foloseste de alocarea dinamica a memoriei acolo unde este necesar.
       Task 2 - Implementarea acestei probleme a durat aproximativ 3 ore. Programul se foloseste de principiile din Task 1 in prinvinta tipului de date neomoge, a functiilor si a parsarii si stocarii continutului. In plus, programul utilizeaza o functie de cautare in continut a cuvantului dat si o serie de functii ajutatoare pentru sortarea site-urilor(swp-interschimbare, cmp-comparare). De asemenea, programul se foloseste de alocarea dinamica a memoriei acolo unde este necesar.
       Task 3 - Implementarea acestei probleme a durat aproximativ 6 ore. Programul se foloseste de principiile din Task 2. In plus, programul foloseste 3 functii de cautare(cautg,cautpoz,cautneg). “cautg” este folosita pentru cautarea in continutul parsat a unei secvente de cuvinte data intre ghilimele. “cautpoz” este folosita pentru cautarea in continutul parsat a unui cuvant dat, iar “cautneg” este folosit pentru cautarea in continutul parsat a unui cuvant dat cu “-”. Programul afiseaza site-urile dupa urmatoarele criterii: daca are in continut secventa data in ghilimele, data are in continut cuvantul dat si daca nu contine cuvantul dat cu “-”, in cazul in care avem mai multe rezultate acestea vor fi sortate crescator lexicografic.
       Task 4 - Implementarea acestei probleme a durat aproximativ 6 ore. Programul se foloseste de principiile din Task 1 in prinvinta tipului de date neomoge, a functiilor si a parsarii si stocarii continutului. In plus, programul se foloseste de 2 functii de rotire a bitilor(rotr si rotl). Aceste functii primesc un numar in baza zecimala si dupa ce il transforma in baza binara ii rotesc biti de k-ori spre dreapta sau stanga. Astfel, este calculat checksum-ul real si comparat cu cel dat.
